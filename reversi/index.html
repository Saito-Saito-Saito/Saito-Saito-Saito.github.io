Stage 0　必要な知識を身につける

先に申し上げておきますと、「ボクまだプログラミングってやったことないの」という方にとって、この内容はハードすぎます。私のリバーシのコードは Python 3 というプログラミング言語で書かれております。理解する上で必要となるプログラミングの知識は

  • 基本的なフロー制御
  • 基本的なリストの制御
  • 基本的な文字列制御
  • 基本的なクラスの扱い

です。どうです、Python かじったことがあれば簡単でしょう。「いや、簡単とかいうなよ、こんなもんド素人でもできるわ。人のことナメすぎやろおめえ」という方は、一度ご自分でリバーシを作ってみてください。そして私のコードと見比べて、「ケッ、やっぱしこいつクズじゃねーか」と優越感に浸る材料にしていただければと思います。逆に「一応プログラミングはやったことあるんだけどなあ、ここら辺あんま覚えてないなあ」という方はぜひこの解説をご覧いただければと思います。

必要となる知識の説明は私よりも巷の参考書やネット記事の方が遥かに優れていますから、ぜひそちらをあたってください。いや、私が説明してもいいかもしれませんけれども、あんま面白くないよ。いやいや、謙遜なんかじゃなくて。そんなに言うなよ、

困っちゃうだろ。

ただ私のコードは私より説明が上手い人がいたら困りますから、その解説をここでやります。いやいや、あんたには無理よ、私のコードをうまく説明なんかできやしねえよ。無理ですって、諦めなさい。

一応備忘録という形である程度のことはこちらに載せてありますが、あくまで「忘れた」人のためのものです。そもそも知らない人はこんな解説じゃなくてしっかりしたので勉強してください。

Stage 1　ルールを把握する

ゲームを作る上でルールを知っているのは大前提です。何より「俺より何年もリバーシやっててメチャクチャ強い人が俺のゲーム使って遊ぶかもしれない」くらいは予想した方がいいかと思います。そんな達人が「おいおい、このゲームろくにルール通りに動きやしねえ」と腹立てて帰ってしまわないように、ルールはしっかり押さえておいてくださいね。

そして必ず一度は、いや一度と言わず何度でも、巷にあふれるリバーシのゲームをやってみてください。そこで「こんなゲームを作ればいいんだ」とイメージを固めておくのは非常に意義があります。

ちなみに言っときますけど、オセロってリバーシの商品名ですよ。Wiki が言ってました。ということでこの解説では「リバーシ」の方を使っていきます。

次回予告

今回は早めに切り上げて、次回はゲームの機能を考えていきます。「こういう機能は必要だろうな」「あんな機能があるといいな」というのを考えておくといいかもしれません。そしてそのままコーディングして、気付いたらリバーシが出来上がってて ... というのが一番理想的なんですけれども、それでは私が仕事したことになりませんから、次回は説明他の人に丸投げしないでちゃんと仕事します。仕事しますから、許して、お願い。お願いしますって。ほら、この通り。
